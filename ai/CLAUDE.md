# Personal Rules

## Code Philosophy
- Duplication > wrong abstraction. Couple unrelated logic only when the benefit is concrete and immediate.
- Idiomatic consistency. Adhere strictly to established framework patterns. Do not introduce alien patterns.
- Safety first. Prioritize type safety, error handling, and security over cleverness.

## Documentation
- If encountering an unfamiliar library, function, or error: consult official docs before writing code. Never guess parameters or behaviour.
- When learning how to use a library or dependency: read official documentation before reading source code.

## Typing
- Explicit over implicit. In typed languages, type hints and return types are mandatory.
